# Open Source Onboarding

This document is for public contributors who want the fastest path into Mandate402 without needing to understand the whole internal operating model first.

## Start Here

1. Read [README.md](../README.md)
2. Read [CONTRIBUTING.md](../CONTRIBUTING.md)
3. Read [CODE_OF_CONDUCT.md](../CODE_OF_CONDUCT.md)
4. Read [docs/README.md](./README.md)
5. Look for a scoped issue
6. Confirm the label and lane of that issue

## Best First Contribution Path

For a first contribution:

- choose a small issue
- avoid auth, infra, contracts, or release-workflow changes
- prefer docs, tests, or isolated UI work unless you already understand the codebase

## Before You Start

- Comment on the issue if you want to work on it.
- Make sure the issue is not already actively owned by someone else.
- Follow the repo workflow and branching rules.

## Where To Ask

If something is unclear:

- ask in the issue
- ask in the PR discussion
- ask through the documented support/contributor guidance paths

## What Success Looks Like

A good first contribution is:

- small
- scoped
- reviewable
- aligned to an issue
- easy to verify

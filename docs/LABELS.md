# Mandatory Label Taxonomy

This document defines the required issue and PR labels for Mandate402.

## Type Labels

- `type:feature`
- `type:bug`
- `type:docs`
- `type:infra`
- `type:security`
- `type:refactor`

## Lane Labels

- `lane:scope`
- `lane:design`
- `lane:frontend-transactional`
- `lane:frontend-observability`
- `lane:backend`
- `lane:web3`
- `lane:ops`

## Priority Labels

- `prio:p0`
- `prio:p1`
- `prio:p2`

## Risk Labels

- `risk:low`
- `risk:medium`
- `risk:high`

## State Labels

- `state:ready`
- `state:blocked`
- `state:in-review`
- `state:needs-design`
- `state:needs-architecture`

## AI Labels

- `ai:allowed`
- `ai:review-required`
- `ai:no-direct-merge`

## Public Onboarding Labels

- `good first issue`
- `help wanted`
- `docs-help`
- `design-help`
- `testing-help`

## Minimum Required Labels Per Issue

Every issue must have at least:

- one `type:*`
- one `lane:*`
- one `prio:*`

High-risk work should also carry a `risk:*` label.

Public-facing contributor issues may also carry one or more onboarding labels from the public onboarding set above.
